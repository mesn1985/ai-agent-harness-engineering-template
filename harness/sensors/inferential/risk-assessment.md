# Risk Assessment

## Question

```text
Could this change create security, data integrity, operational, API compatibility, or user-facing risk?
```

## Checklist

```text
- Authentication and authorization impact considered.
- Data creation, modification, deletion, or migration impact considered.
- Public API compatibility considered.
- Performance impact considered.
- Operational and deployment impact considered.
- Rollback or mitigation considered if relevant.
```

## Escalate If

```text
- Risk is medium or higher and not explicitly authorized.
- The change affects security-sensitive behavior.
- The change could cause data loss.
- The change affects production deployment or rollback.
```
